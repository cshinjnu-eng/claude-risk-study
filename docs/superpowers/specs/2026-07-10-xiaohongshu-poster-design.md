# Claude 封号调查小红书海报设计规格

## 目标

制作 4 张 1080 × 1350 px 小红书图文海报，面向没有统计学背景的 Claude 中文用户。海报负责三件事：说明目前最清晰的风险线索、纠正常见传言、引导读者进入测评网站。

海报只陈述样本内结果。不得把相关性写成平台规则，不得把组内异常比例写成个人封号概率。

## 阅读路径

1. 先用常见传言吸引读者，但封面直接给出克制结论。
2. 第二页只解释目前最稳定的具体风险方向。
3. 第三页展示较少报告异常的受访者画像，并和模型结果分开。
4. 最后一页完成辟谣、方法披露和测评转化。

## 页面内容

### 01 封面：传言经过检验以后

- 主标题：`中文、Claude Code、高用量，真的会导致封号吗？`
- 结论：`目前没有稳定证据。`
- 支撑：`累计 315 份问卷；249 个账号进入每日风险模型。`
- 行动：显示网站地址和小型二维码。
- 画面：人物面对三条传言，红色校验线穿过画面，形成由传言到证据的阅读方向。

### 02 主要发现：先检查连接状态

- 主标题：`目前最清晰的线索，是代理可能在使用中失效。`
- 解释：回答“可能发生过”的账号，当天发生异常的相对风险约为参照组的 2.18 倍。
- 必须紧邻数字说明：`2.18 倍是两组间的相对比较，不是个人封号概率。`
- 次要线索：账号使用 1–6 个月的风险方向较稳定；新加坡节点仍属初步线索；机构网络样本太少，暂不下结论。
- 画面：连续网络路径在中段断开；2.18 是全页唯一超大数字。
- 行动：底部显示网站地址和小型二维码。

### 03 描述画像：哪些人较少报告异常

- 页首必须标注：`描述性统计，不是模型结论。`
- 展示四组：
  - 固定少数人共用节点：32%，n=19。
  - 本人银行卡或本人长期支付账号：43%，n=23。
  - 机房或数据中心 IP：47%，n=93。
  - 账号使用 6–12 个月：48%，n=33。
- 背景参照：当前自愿样本整体有 62% 报告异常，异常用户被过度代表。
- 结论：这些比例用来描述样本，不是保号方法。
- 画面：四条不等长的实体色带或四个人物侧影，避免四张重复卡片。
- 行动：底部显示网站地址和小型二维码。

### 04 辟谣、方法与入口

- 主标题：`这些说法，目前都没有稳定证据。`
- 辟谣对象：中文或混合语言、Claude Code 或其他使用入口、高用量、Google 绑定手机号。
- Google 手机号需标注：有效回答较少，暂不下结论。
- 方法数字：315 份累计问卷；249 个账号；1,872 个风险日；57 个事前问题统一扫描。
- 边界：独立调查，与 Anthropic 无隶属关系；样本内关联不等于官方封号规则。
- 行动：大型二维码、网站地址 `http://claude-risk.bsbsanwu.xyz/`、文案 `11 题，约 90 秒。`

## 视觉合同

- 画布：1080 × 1350 px，四周安全边距 76 px。
- 风格：中文编辑部调查版面，克制、直接、有纸张质感，不做科技产品落地页。
- 配色：米白 `#F3F0E7`、墨黑 `#161614`、证据红 `#F04438`、次要灰 `#77766F`。不用蓝紫渐变、玻璃拟态或发光。
- 字体：中文使用清晰的黑体系统；英文数字使用等宽字体。标题、正文、注释至少形成三级字号差。
- 构图：解释文字默认左对齐。每页只允许一个主视觉锚点；使用线条、留白和对齐分组，不使用卡片墙。
- 节奏：封面为 anchor，第二页为 dense，第三页为 structured，末页为 breathing。
- 图像：使用编辑式人物剪影、网络路径和统计色带。图像辅助阅读，不承担未经数据支持的叙事。
- 二维码：每页二维码均指向测评网站；前三页为小入口，第四页为主要入口。必须使用真实二维码，不留占位框。

## 文案规则

- 用“报告异常”“相对风险”“目前没有稳定证据”，避免“必封”“保护作用”“安全账号”。
- 不在正文堆叠 OR、CI、q 值。第二页保留 OR 2.18 的通俗解释；完整区间留在网站。
- 不使用“重磅”“真相来了”“震惊”等炒作词。
- 不用三段式口号和否定式排比。句子长短应有变化。

## 生成与验证

- 来源文件为单个 HTML/CSS 海报画布，四页作为独立 section，生成 PNG 时逐页截图。
- 生成二维码并嵌入源文件，扫码验证目标地址。
- 检查每页 1080 × 1350、文字无裁切、最小字号可读、颜色对比足够。
- 视觉检查覆盖四页整体节奏，避免重复版式、重复卡片和信息拥挤。
- 最终输出 `01-cover.png` 至 `04-method-and-qr.png`，并更新 GitHub 仓库。

## 验收标准

- 用户在封面能看懂主题、当前结论和入口。
- 每页只有一个主要结论，统计数字旁有必要的解释边界。
- 第三页不会被误读为保护因素排行榜。
- 四页均能进入测评网站，最后一页二维码可扫描。
- 页面源文件和 PNG 均纳入版本控制，渲染结果经过桌面视觉检查。
